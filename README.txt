Run command:
npm i

Then you can run the tests to see the output. You run the tests with this command:
npm test

You can click on the links in the output to see which line a failure or error happened on.

There is an answers folder that contains possible solutions for all of the problems.
Look in there if you get sick of working on it and want to see how it's done.
I didn't "dumb down" the answers. I wrote it the same way I would at work,
so there might be some advanced stuff in there. Look up "ES6" for help figuring out
what that stuff is. Specifically, you might want to look up "destructuring" and "spread"
